01

¿Qué es diseño responsive?

Es la técnica de hacer que un sitio web se adapte automáticamente al tamaño de la pantalla donde se visualiza — sin hacer páginas separadas para móvil y desktop.

Antes se hacían dos versiones del sitio: m.sitio.com para móvil y sitio.com para desktop. Hoy, con CSS, un solo archivo se adapta a todo.

📱
Mobile
360px
1 columna, menú hamburguesa
📟
Tablet
768px
2 columnas, nav compacta
💻
Laptop
1024px
3–4 columnas, layout completo
🖥️
Desktop
1440px+
Diseño máximo, sidebars
02

La etiqueta viewport — OBLIGATORIA

Esta línea va en el <head> de todo sitio web responsive. Sin ella, el móvil simula una pantalla de 980px y todo se ve miniatura.

index.html 
<!-- ✅ Siempre incluye esto en tu <head> -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- ¿Qué significa cada parte? -->
<!-- width=device-width  → el ancho = el ancho real del dispositivo -->
<!-- initial-scale=1.0   → sin zoom al cargar -->
⚠️

Sin esta meta tag, tus media queries no funcionarán correctamente en móviles. Es lo primero que debes verificar si algo falla.

03

Sintaxis de @media

Un media query se escribe al final de tu CSS y "sobreescribe" las reglas anteriores cuando la condición se cumple.

style.css — Estructura básica 
/* 1. Estilo base (para desktop) */
.caja {
  width: 800px;
  font-size: 24px;
  background: blue;
}

/* 2. Media query: se aplica solo si la pantalla ≤ 768px */
@media (max-width: 768px) {
  .caja {
    width: 100%;     /* ahora ocupa todo el ancho */
    font-size: 16px; /* letra más pequeña */
  }
  /* solo cambias lo que necesita cambiar */
  /* background: blue sigue igual, no lo repites */
}
style.css — Tipos de condiciones 
/* max-width: se activa en pantallas MENORES O IGUALES a X */
@media (max-width: 768px) { ... }

/* min-width: se activa en pantallas MAYORES O IGUALES a X */
@media (min-width: 1024px) { ... }

/* rango: entre 600px y 900px */
@media (min-width: 600px) and (max-width: 900px) { ... }

/* orientación: pantalla horizontal (landscape) */
@media (orientation: landscape) { ... }

/* preferencia de movimiento reducido (accesibilidad) */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; }
}
💡

Regla de oro: los media queries siempre van al final de tu archivo CSS, después de todos los estilos base. CSS lee de arriba hacia abajo, así que el último valor gana.

04

Breakpoints más usados

Un breakpoint es el punto exacto (en píxeles) donde tu diseño "cambia de forma". No hay regla oficial, pero estos son los más comunes:

Breakpoint Dispositivo Uso típico Tipo
480px Teléfonos pequeños 1 columna, todo vertical small
768px Tablets / móviles grandes 2 columnas, menú colapsado mobile
1024px Tablets grandes / laptops Transición desktop ↔ móvil tablet
1280px Laptops estándar Layout completo desktop
1440px+ Monitores grandes max-width en contenedor wide
🌿

Tip: no copies estos números mecánicamente. Abre tu sitio, achica la ventana del navegador, y pon el breakpoint donde el diseño "se rompe". Ese es tu breakpoint real.

05

Mobile First vs Desktop First

Son dos filosofías opuestas. La diferencia está en cuál es tu CSS base y qué tipo de media query usas.

❌ Desktop First (la que ya tienes)
/* base: estilos para desktop */
.grid {
  grid-template-columns: repeat(4, 1fr);
}

/* luego "achicas" para móvil */
@media (max-width: 768px) {
  .grid {
    grid-template-columns: 1fr;
  }
}
✅ Mobile First (recomendado)
/* base: estilos para móvil */
.grid {
  grid-template-columns: 1fr;
}

/* luego "expandes" para desktop */
@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(4, 1fr);
  }
}

Mobile First es el estándar moderno porque más del 60% del tráfico web viene de móviles. Pero para un proyecto personal como el tuyo, Desktop First está perfectamente bien — es más fácil de aprender al inicio.

06

Unidades relativas — el secreto del responsive

Usar px fijos es el enemigo del diseño responsive. Estas unidades se adaptan solas:

Unidad Relativa a Ejemplo Uso ideal
% El elemento padre width: 80% Anchos de contenedores
rem Font-size del <html> (16px default) 1rem = 16px Tipografía y espaciados
em Font-size del elemento padre 1.5em Padding relativo al texto
vw 1% del ancho del viewport 50vw = mitad de pantalla Títulos grandes, hero sections
vh 1% del alto del viewport 100vh = pantalla completa Secciones full screen
dvh Alto dinámico (mobile real) 100dvh Móviles con barra de navegador
Ejemplo práctico 
/* ❌ Valores fijos — no se adaptan */
.titulo { font-size: 80px; }
.caja   { width: 1200px; }

/* ✅ Valores relativos — se adaptan solos */
.titulo { font-size: 5vw; }        /* 5% del ancho de pantalla */
.caja   { max-width: 90%; }        /* nunca más del 90% del padre */
.boton  { padding: 0.8rem 1.5rem; } /* escala con la fuente base */
07

Flexbox & Grid — los mejores aliados

Estas dos propiedades de CSS ya tienen comportamiento responsive incorporado.

Flexbox — wrap automático 
.nav {
  display: flex;
  flex-wrap: wrap;  /* si no caben, bajan a la siguiente línea */
  gap: 16px;
}

/* Sin media query, el nav se adapta solo */
/* En desktop: todos en una fila */
/* En móvil:   se envuelven en 2–3 filas */
CSS Grid — auto-fill mágico 
.cards {
  display: grid;
  /* auto-fill: crea tantas columnas como quepan */
  /* minmax(250px, 1fr): cada columna mín 250px, máx flexible */
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 16px;
}

/* Resultado: */
/* Desktop (1200px):  4 columnas automáticamente */
/* Tablet  (768px):   3 columnas automáticamente */
/* Mobile  (360px):   1 columna automáticamente  */
/* ¡Sin ningún media query! */
✨

Pro tip: repeat(auto-fill, minmax(250px, 1fr)) es probablemente la línea de CSS más poderosa para grids de cards. Apréndetela de memoria.

08

La función clamp() — tipografía fluida

En vez de cambiar el font-size en cada media query, clamp() lo hace escalar fluidamente entre un mínimo y un máximo.

Sintaxis de clamp() 
/*  clamp( MÍNIMO,  IDEAL,  MÁXIMO ) */
.titulo {
  font-size: clamp(2rem, 8vw, 5rem);
}

/* En mobile  (360px):  2rem  = 32px  ← no baja de aquí */
/* En tablet  (600px):  8vw   = 48px  ← escala con la pantalla */
/* En desktop (900px):  5rem  = 80px  ← no sube de aquí */

/* Equivalente SIN clamp (más código, menos elegante): */
.titulo { font-size: 5rem; }
@media (max-width: 768px) { .titulo { font-size: 3rem; } }
@media (max-width: 480px) { .titulo { font-size: 2rem; } }

// Visualización: clamp(2rem, 8vw, 5rem)

09

Imágenes responsive

Las imágenes con ancho fijo se salen de la pantalla en móvil. La solución es simple:

Regla base — applica a TODAS las imágenes 
/* Pon esto siempre en tu CSS global */
img {
  max-width: 100%;  /* nunca más ancho que su contenedor */
  height: auto;     /* mantiene la proporción original */
  display: block;   /* elimina el espacio fantasma debajo */
}
object-fit — para imágenes con altura fija 
.foto-perfil {
  width: 300px;
  height: 300px;
  object-fit: cover;   /* recorta para llenar, sin distorsionar */
  border-radius: 50%;  /* círculo */
}

/* Valores de object-fit: */
/* cover   → rellena el espacio, recorta si es necesario ✅ más usado */
/* contain → imagen completa, puede dejar espacios vacíos */
/* fill    → distorsiona para llenar el espacio ❌ evitar */
10

Checklist final — antes de publicar

Antes de dar por terminado cualquier sitio, verifica estos puntos:

  • Tienes <meta name="viewport" content="width=device-width, initial-scale=1.0"> en tu HTML
  • Las imágenes tienen max-width: 100% y height: auto
  • No usas anchos fijos en px para contenedores principales (usa %, max-width, o vw)
  • Tus media queries están al final del CSS, ordenadas de mayor a menor (si usas Desktop First)
  • Probaste en Chrome DevTools con el modo responsive (F12 → ícono de móvil)
  • El texto es legible sin hacer zoom en móvil (mínimo 14–16px)
  • Los botones son fáciles de tocar en móvil (mínimo 44×44px de área táctil)
  • El header/nav no se sale de la pantalla en 360px de ancho
  • No hay scroll horizontal en ningún breakpoint
  • Usas clamp() o media queries para títulos con font-size muy grandes
🛠️

Herramienta esencial: En Chrome, presiona F12 → haz clic en el ícono de móvil arriba a la izquierda. Puedes simular cualquier dispositivo y ver exactamente cómo se ve tu sitio en cada resolución.
